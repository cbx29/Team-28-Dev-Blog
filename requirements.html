Initial Project Objectives

A list of the key criteria that our end-product should meet:

  • Detect and track gestures for atleast 2 players
  • Develop a range of demonstratable game examples
  • Added functions to the existing MFC settings panel.
  • Generate an energy report from data collected during the game.

Q1: Who are our end users?

Schools with kids in games (e.g. webcam on wall, tv or projector on wall showing a game, 2 or more children playing), wards with sick children in a hospital who need exercise for rehab, special needs clinics with children who need to play with others, elderly in care homes playing with their friends, friends at home playing games for exercises in fitness competitions.

Q2: What is the end product that we need to create?
  • A new settings panel or added functions to the existing PseudoVR, facial navigation and multitouch MFC settings panel.
  • A game engine with Wii-sports like games.
  • Demonstratable game examples.
  • API for adding more motion games. E.g. archery, skiing
Q3: What goals do you expect from the end product?

Population health, for everyone. Including those with accessibility needs. Social engagement with others through playing games. Smooth multiplayer experiences. Simple, easy to use navigation from start to finish, crash resilience and knowing what level of hardware is cheapest to run effectively (most efficient). Reporting back to intel with the performance reports for their testing teams.
